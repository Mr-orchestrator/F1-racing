F1 Racing Store · racing-f1-rho.vercel.app · Org D1D7123F524450A60A490D45@AdobeOrg

Overview & Status

Move from a single-solution AppMeasurement implementation (Adobe Analytics only, via the legacy 2o7.net beacon) to a unified Adobe Web SDK collection that powers four solutions from one event: Adobe Analytics, Customer Journey Analytics (CJA), Real-Time CDP (RTCDP), and edge personalization. Collect once, distribute everywhere.

AA RSID (prod) ageo1xxlonf1racingstore-prod Base currency INR On-site prices USD Time zone IST Source layer window.adobeDataLayer
✅ Live today: AppMeasurement page-view → Adobe Analytics (verified HTTP 200, cc=USD). Page View rule + currency data element published in Tags library a4.
📋 Planned (this plan): Web SDK, XDM schema, CJA, and RTCDP are fully specified but not yet built in Adobe. They are implemented in the Adobe Launch (Tags) + AEP UIs by following the Runbook below.
Why migrate off 2o7.net: (1) it does not feed AEP, so CJA & RTCDP get nothing; (2) the 2o7.net host is heavily ad-blocked → visit/revenue undercount. The Web SDK edge domain (*.adobedc.net) fixes both.

PII rule: email is hashed client-side (SubtleCrypto SHA-256) before it ever reaches the Edge. Raw PII is never sent. Merge policy (RTCDP): private graph, timestamp-ordered, ECID+crmId stitched.

XDM - productListItems (custom code)

var dl = window.adobeDataLayer || [];
var last = dl[dl.length - 1] || {};
var items = (last.cart && last.cart.item) ? last.cart.item : [];
return items.map(function (it) {
  var info = it.productInfo || {};
  var price = (it.price && (it.price.amount != null ? it.price.amount : it.price)) || info.price || 0;
  return {
    SKU: info.productID || it.product_id || '',
    name: info.productName || it.product_name || '',
    quantity: Number(info.quantity || it.quantity || 1),
    priceTotal: Number(price) * Number(info.quantity || 1),
    currencyCode: 'USD',
    productCategories: it.category ? [{ categoryID: it.category }] : []
  };
});

Runbook · 3 — Tags / Web SDK

  • Install Adobe Experience Platform Web SDK extension. Datastreams per environment: Dev→Dev ID, Staging→Dev ID, Production→Prod ID. Edge domain default *.adobedc.net.
  • Create all DL - * data elements + Email - SHA256 + XDM - productListItems + XDM - ExperienceEvent (see SDR).
  • Create one rule per event: Event = ACDL Data Pushed with condition %DL - event% = "<name>" → Action 1 Web SDK Set variable (XDM) → Action 2 Web SDK Send event.
  • Min P0 rules: Page View, Product viewed, Add to cart, Remove from cart, BeginCheckout, Purchase.
  • New library → Add All Changed Resources → Build to Dev → validate → Submit → Approve → Publish to Production.

XDM - ExperienceEvent (custom code, abbreviated)

var dl = window.adobeDataLayer || [];
var e = dl[dl.length - 1] || {}, a = e.attributes || {};
var xdm = { eventType: 'web.webpagedetails.pageViews',
  web:{ webPageDetails:{ name:_satellite.getVar('DL - pageName'), pageViews:{ value:1 } } },
  identityMap:{} };
if (_satellite.getVar('DL - userId')) xdm.identityMap.crmId=[{id:String(_satellite.getVar('DL - userId')),primary:true}];
switch (e.event) {
  case 'Add to cart':
    xdm.eventType='commerce.productListAdds';
    xdm.commerce={ productListAdds:{ value:1 } };
    xdm.productListItems=[{ SKU:a.product_id, name:a.product_name, quantity:Number(a.product_quantity)||1, priceTotal:Number(a.product_price)||0, currencyCode:a.currency||'USD' }];
    break;
  case 'Purchase': case 'Purchase completed':
    xdm.eventType='commerce.order';
    xdm.commerce={ purchases:{ value:1 }, order:{ purchaseID:String(a.transaction_id||''), priceTotal:Number(a.transaction_total)||0, currencyCode:a.currency||'USD' } };
    xdm.productListItems=_satellite.getVar('XDM - productListItems');
    break;
  // … other events map to commerce.* or web.webInteraction + _f1racingstore.*
}
return xdm;

How it's implemented (important)

This document is the blueprint. Nothing here is auto-deployed to Adobe. All of it is built by hand in the Adobe UIs — exactly like you built the Page View rule:
LayerBuilt in
XDM schemas, datasets, identities, merge policyAEP (Experience Platform)
Datastream (AA + AEP services)Data Collection → Datastreams
Web SDK extension, XDM data element, rules, sendEventAdobe Launch / Tags
Connection, Data View, dashboardsCJA
Audiences, destinationsRTCDP

The xdm/*.json files are reference blueprints — you recreate that structure in AEP Schema Builder (they aren't importable). Automation is possible via the Reactor API (Tags) and Schema Registry API (AEP) if you create an Adobe Developer Console service credential.
